## Data Relationships

### Hospital ↔ Beds
```
1 Hospital → Many Beds
Relationship: hospitalCode
Example: HOSP123456 contains 250 beds
```

### Hospital ↔ Patients
```
1 Hospital → Many Patients
Relationship: hospitalCode
```

### Beds ↔ Patients
```
1 Bed ←→ 1 Patient (when occupied)
Primary Relationship: bedId in patients collection
Secondary Relationship: patientId in beds collection

When patient admitted:
- patients.bedId = beds._id
- beds.patientId = patients._id

When patient discharged:
- beds.patientId = null
- Bed becomes available for next patient
```

## Query Patterns

### 1. Get All Available Beds in Ward
```javascript
const q = query(
  collection(db, "beds"),
  where("hospitalCode", "==", "HOSP123456"),
  where("ward", "==", "Ward-1"),
  where("status", "==", "available")
);
const snapshot = await getDocs(q);
```

### 2. Get All Occupied Beds
```javascript
const q = query(
  collection(db, "beds"),
  where("hospitalCode", "==", "HOSP123456"),
  where("status", "==", "occupied")
```

## Data Integrity Rules

### Bed Creation
```
Rule: All beds must have:
- hospitalCode ✓
- ward (format: "Ward-N")
- room (format: "Room-N")
- bedNumber (1-10)
- type ("regular" or "ICU")
- status ("available" initially)
- patientId (null initially)

Total beds = numberOfWards × roomsPerWard × (bedsPerRoom + icuBedsPerRoom)
```

### Patient Admission
```
Rule: Before creating patient:
1. Verify bed exists and status == "available"
2. Verify patient not already admitted
3. Lock bed and patient creation together

Updates:
- Patient created with bedId reference
- Bed status changed to "occupied"
- Bed patientId set to patient's ID
```

### Patient Discharge
```
Rule: When discharging:
1. Mark patient discharged = true
2. Set dischargeDate = now()
3. Set bed status = "available"
4. Clear bed patientId = null
5. DO NOT delete any records
```

## Important Notes

1. **No Deletion:** Never delete bed or patient records. Just mark as discharged.

2. **Consistent References:** Always maintain bidirectional references:
   - Patient.bedId → Bed._id
   - Bed.patientId → Patient._id

3. **Status Values:**
   - Beds: "available" | "occupied"
   - Patients: "discharged" (boolean)
   - Staff: "pending" | "approved"

4. **Hospital Code:** Critical for data isolation. Always filter by hospitalCode.

5. **Timestamps:** Use serverTimestamp() for accuracy across timezones.

6. **Backward Compatibility:** Existing fields remain unchanged. New fields are additive.
